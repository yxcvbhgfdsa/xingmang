
# 星芒AI写作

## 本写作项目基于星盘基座0.10版本魔改而成

## 🚀 快速启动

### 方式一：一键启动（推荐）
前提是已经手动安装nodejs
双击运行 `start.bat` 文件，脚本会自动：
- 检查Node.js环境
- 安装所有依赖
- 启动后端服务（端口3000）
- 启动前端服务（端口5173）
加载会有些慢

### 方式二：手动启动
```bash
# 1. 安装依赖
npm run install-all

# 2. 启动后端
cd server
npm start

# 3. 启动前端（新开终端）
cd novel
npm run dev
```

## 📱 访问地址

- **前端界面**: http://localhost:5173
- **后端API**: http://localhost:3000

## 🛠️ 功能特性

- ✅ **书本管理**: 创建、编辑、删除小说
- ✅ **创意工坊**:固定的模板生成，可固定提示词以及匹配占位符字段
- ✅ **工作流**:链状工作流，目前仅仅是能用，但还需改善
- ✅ **经验分享**:用于编写个人经验，方便分享，温故而知新
- ✅ **AI续写**: 智能续写功能，支持流式输出
- ✅ **提示词管理**: 自定义AI提示词
- ✅ **备忘录**: 跨书本共享的创作素材
- ✅ **API配置**: 支持多种AI服务商
- ✅ **对话历史**: 完整的聊天记录管理

## 🔧 技术栈

- **前端**: Vue3 + Element Plus + TypeScript
- **后端**: Node.js + Express
- **数据库**: SQLite
- **AI集成**: 支持OpenAI、Claude、通义千问等

## 📝 常见问题

**Q: 启动失败怎么办？**
A: 确保已安装Node.js，然后运行 `start.bat` 重新安装依赖

**为什么浏览器不显示？**
A:端口占用或者太老了，最好换个浏览器，如果还不行，那就是端口占用，下一版我会改

**API配置是怎样的？**
A:选择其他服务商，使用open ai兼容接口，记住，不是open AI服务商

A: 修改 `server/index.js` 和 `novel/vite.config.ts` 中的端口号"# -"
