
本写作项目基于星盘基座0.10版本魔改而成

# 网文创作辅助系统

## 🚀 快速启动

### 方式一：一键启动（推荐）
双击运行 `start.bat` 文件，脚本会自动：
- 检查Node.js环境
- 安装所有依赖
- 启动后端服务（端口3000）
- 启动前端服务（端口5173）
加载会有些慢
### 方式二：快速启动
如果已经安装过依赖，可以双击运行 `quick-start.bat` 快速启动服务。

### 方式三：手动启动
```bash
# 1. 安装依赖
npm run install-all

# 2. 启动后端
cd server
npm start

# 3. 启动前端（新开终端）
cd novel
npm run dev
```

## 📱 访问地址

- **前端界面**: http://localhost:5173
- **后端API**: http://localhost:3000

## 🛠️ 功能特性

- ✅ **书本管理**: 创建、编辑、删除小说
- ✅ **创意工坊**:固定的模板生成，可固定提示词以及匹配占位符字段
- ✅ **工作流**:链状工作流，目前仅仅是能用，但还需改善
- ✅ **经验分享**:用于编写个人经验，方便分享，温故而知新
- ✅ **AI续写**: 智能续写功能，支持流式输出
- ✅ **提示词管理**: 自定义AI提示词
- ✅ **备忘录**: 跨书本共享的创作素材
- ✅ **API配置**: 支持多种AI服务商
- ✅ **对话历史**: 完整的聊天记录管理

## 🔧 技术栈

- **前端**: Vue3 + Element Plus + TypeScript
- **后端**: Node.js + Express
- **数据库**: SQLite
- **AI集成**: 支持OpenAI、Claude、通义千问等

## 📝 使用说明

1. 首次使用需要配置API密钥（在"API配置"页面）
2. 创建书本后可以开始创作
3. 使用 `user: 你的指令` 格式进行AI续写
4. AI会以 `-> 内容 <-` 格式回复

## 🆘 常见问题

**Q: 启动失败怎么办？**
A: 确保已安装Node.js，然后运行 `start.bat` 重新安装依赖

**Q: 如何停止服务？**
A: 关闭对应的服务窗口即可

**Q: 端口被占用怎么办？**
A: 修改 `server/index.js` 和 `novel/vite.config.ts` 中的端口号"# -"
